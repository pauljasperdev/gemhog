**Important context:** E2E tests have a pre-existing environment issue where Playwright gets stuck during baseline verification (documented in STATE.md). **Also important:** The cookie consent banner only shows when `NEXT_PUBLIC_POSTHOG_KEY` is set AND PostHog's consent status is "pending". In E2E with no PostHog key configured, the banner won't appear. The test must account for this:

Test structure:
```typescript
import { expect, test } from "./fixtures";

test.describe("Cookie consent banner", () => {
  test("banner appears on first visit when PostHog is configured", async ({ page }) => {
    // Skip if PostHog key not configured in E2E environment
    await page.goto("/");

    // Check if banner appears (depends on NEXT_PUBLIC_POSTHOG_KEY being set)
    const banner = page.getByRole("dialog", { name: "Cookie consent" });

    // If PostHog is not configured, banner won't appear -- this is expected
    const bannerVisible = await banner.isVisible().catch(() => false);
    if (!bannerVisible) {
      test.skip();
      return;
    }

    // Banner content
    await expect(page.getByText("Would you like a cookie?")).toBeVisible();
    await expect(page.getByRole("button", { name: "Accept" })).toBeVisible();
    await expect(page.getByRole("button", { name: "Decline" })).toBeVisible();
  });

  test("accept dismisses banner", async ({ page }) => {
    await page.goto("/");
    const banner = page.getByRole("dialog", { name: "Cookie consent" });
    const bannerVisible = await banner.isVisible().catch(() => false);
    if (!bannerVisible) {
      test.skip();
      return;
    }

    await page.getByRole("button", { name: "Accept" }).click();
    await expect(banner).not.toBeVisible();
  });

  test("decline dismisses banner", async ({ page }) => {
    await page.goto("/");
    const banner = page.getByRole("dialog", { name: "Cookie consent" });
    const bannerVisible = await banner.isVisible().catch(() => false);
    if (!bannerVisible) {
      test.skip();
      return;
    }

    await page.getByRole("button", { name: "Decline" }).click();
    await expect(banner).not.toBeVisible();
  });
});
```

The tests gracefully handle environments where PostHog is not configured by skipping, rather than failing. This is appropriate because the E2E environment may not have PostHog keys configured, and the unit tests in Task 1 already provide thorough coverage of the component logic.
