<task type="auto">
  <name>Task 2: Add font-display Tailwind utility via CSS theme</name>
  <files>
    apps/web/src/index.css

  <action>
Use the `/web-design-guidelines` skill for font loading best practices and Tailwind CSS theme configuration guidance.

Add the DM Serif Display font-family to the Tailwind theme in `apps/web/src/index.css`:

1. In the `@theme inline` block, add:
   ```
   --font-display: var(--font-dm-serif), "DM Serif Display", serif;
   ```
   This creates the `font-display` Tailwind utility class (used as `className="font-display"`)

2. The `--font-dm-serif` CSS variable is set by next/font/google on the `<html>` element (from Task 1). The fallback ensures the font name resolves even if the variable isn't loaded yet.

**Why this approach:** Matches the existing pattern where `--font-sans` is defined in `@theme inline`. Tailwind v4 reads `--font-*` variables from `@theme` to generate font utilities.

**Do NOT:**
- Add a separate `@font-face` declaration (next/font handles this)
- Modify the `:root` or `.dark` CSS variable blocks (font is theme-independent)
