**1. Create SignupForm client component**
(`apps/web/src/components/signup-form.tsx`):

- Add `"use client"` directive
- Import `useMutation` from `@tanstack/react-query`, `useForm` from
  `@tanstack/react-form`, `z` from `zod`, `trpc` from `@/trpc/client`,
  `AnalyticsEvents` and `trackEvent` from `@/lib/analytics`
- Create named export `SignupForm` (not default -- follows shadcn pattern)
- Set up tRPC mutation:
  `const subscribe = useMutation(trpc.subscriber.subscribe.mutationOptions())`
- Set up form with `useForm`:
  - `defaultValues: { email: "" }`
  - `validators: { onSubmit: z.object({ email: z.email("Please enter a valid email address") }) }`
  - `onSubmit: async ({ value })` handler that:
    1. Calls `trackEvent(AnalyticsEvents.SIGNUP_STARTED)`
    2. Calls `await subscribe.mutateAsync(value)`
    3. Calls `trackEvent(AnalyticsEvents.SIGNUP_COMPLETED)`
- **Success state:** When `subscribe.isSuccess`, render confirmation message:
  `<p className="text-emerald-400" role="status">Check your inbox to confirm your subscription.</p>`
  (Use `role="status"` for accessibility -- screen readers announce this)
- **Error state:** Show tRPC error message below form. Use
  `subscribe.error?.message` with fallback text. Display as
  `<p role="alert" className="text-red-400 text-sm mt-2">...</p>`
- **Form layout:**
  - Wrap in
    `<form onSubmit={(e) => { e.preventDefault(); e.stopPropagation(); form.handleSubmit(); }}>`
    (match sign-in-form pattern)
  - Use `form.Field` for email input with `name="email"`
  - Inline layout: `flex flex-col sm:flex-row gap-3` (stacked on mobile, inline
    on desktop)
  - Email input: Use `<input>` (not shadcn Input -- we need custom dark
    styling). Style with:
    `type="email" placeholder="your@email.com" className="flex-1 rounded-lg border border-gray-700 bg-white/5 px-4 py-3 text-white placeholder:text-gray-500 focus:border-emerald-500 focus:outline-none focus:ring-1 focus:ring-emerald-500"`
    plus `aria-label="Email address"` for accessibility
  - Submit button: Use `<button>` with emerald styling:
    `className="rounded-lg bg-emerald-500 px-6 py-3 font-medium text-white transition-colors hover:bg-emerald-400 focus:outline-none focus:ring-2 focus:ring-emerald-500 focus:ring-offset-2 focus:ring-offset-gray-950 disabled:opacity-50 disabled:cursor-not-allowed sm:whitespace-nowrap"`
  - Use `form.Subscribe` to access `state.canSubmit` and `state.isSubmitting`
  - Button text: "Get the free newsletter" (default), "Subscribing..." (when
    submitting)
  - Disable button when `!state.canSubmit || state.isSubmitting`
- **Validation errors:** Show below the input using `field.state.meta.errors`
  mapping, styled as `text-red-400 text-sm`
- **Privacy consent text below form:** Small text below the form with privacy
  policy link. Something like:
  `"By subscribing, you agree to receive our newsletter. Unsubscribe anytime."`
  followed by a
  `<Link href="/privacy" className="underline hover:text-gray-300">Privacy Policy</Link>`.
  Style: `text-gray-500 text-xs mt-3`. The exact wording is at Claude's
  discretion per CONTEXT.md but must be friendly tone, mention privacy policy
  link, and clearly indicate what user is signing up for.
