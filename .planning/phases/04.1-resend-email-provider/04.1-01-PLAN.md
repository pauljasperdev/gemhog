       ```bash
       pnpm add resend --filter @gemhog/core
       pnpm remove @aws-sdk/client-sesv2 --filter @gemhog/core
       ```

    2. Rewrite `packages/core/src/email/email.service.ts`:
       - Remove the `@aws-sdk/client-sesv2` import entirely
       - Add `import { Resend } from "resend"` and `import { Schedule } from "effect"` (add to existing effect import)
       - Keep `SendEmailParams`, `EmailService`, `EmailServiceTag`, and `EmailServiceConsole` EXACTLY as they are -- do NOT change the interface or the console implementation
       - Replace `makeEmailServiceLive` factory:
         - Change signature from `(senderEmail: string)` to `(apiKey: string, fromEmail: string)`
         - Create `new Resend(apiKey)` once inside `Layer.sync` (NOT per send call)
         - In the `send` method, call `resend.emails.send({ from: fromEmail, to: [to], subject, html, headers })` -- note `to` is an array
         - Resend returns `{ data, error }` instead of throwing. Destructure and `throw error` if error is truthy, inside the `tryPromise.try` block
         - Keep `catch` mapping to `new EmailSendError({ message: ..., cause: error })`
         - Add retry for transient errors: wrap the send Effect with `Effect.retry({ schedule: retrySchedule, while: (error) => isTransientError(error) })`
         - Use `Schedule.exponential("500 millis").pipe(Schedule.compose(Schedule.recurs(3)))` for the retry schedule
         - End the pipeline with `Effect.asVoid` to match the `void` return type
       - Add an `isTransientError` helper function (NOT exported):
         - Takes `EmailSendError`, inspects `error.cause` for Resend error names
         - Transient names: `"rate_limit_exceeded"`, `"daily_quota_exceeded"`, `"monthly_quota_exceeded"`, `"application_error"`, `"internal_server_error"`
         - Also check if `cause.statusCode >= 500`
         - Return `false` if cause is undefined or doesn't match

    3. No changes needed to `email.errors.ts` -- the existing `EmailSendError` with `cause?: unknown` already captures Resend errors. The `isTransientError` helper inspects `cause` at runtime.
  </action>
