## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Email sending failed in local dev with placeholder Resend API key**

- **Found during:** Task 1 verification
- **Issue:** After Plan 01 made RESEND_API_KEY required, `localDevServerEnv` provides `"re_local_dev_placeholder"` which passes Zod validation (`startsWith("re_")`) but is not a real Resend API key. The `makeEmailLayers` factory always created a live Resend client, causing `EmailSendError` on every subscribe call during integration tests.
- **Fix:** Plan 01 (running in parallel) incorporated the fix into `packages/core/src/email/email-layers.ts`: added `DEV_PLACEHOLDER_KEY = "re_local_dev_placeholder"` constant and conditional that uses `EmailServiceConsole` when the placeholder key is detected.
- **Files modified:** `packages/core/src/email/email-layers.ts` (committed as part of Plan 01's commit `196bff1`)

**2. [Rule 3 - Blocking] Type error in makeEmailLayers prevented Next.js builds**

- **Found during:** Task 1 verification
- **Issue:** `makeEmailLayers` parameter `databaseLive: Layer.Layer<any>` defaulted to `Layer.Layer<any, never, never>`, but `DatabaseLive` has `SqlError` in its error channel (`Layer.Layer<..., SqlError, never>`). This caused TypeScript compilation failures blocking `pnpm build`.
- **Fix:** Changed type to `databaseLive: Layer.Layer<any, any>` to accept error channel.
- **Files modified:** `packages/core/src/email/email-layers.ts` (committed as part of Plan 01's commit `196bff1`)

**Note:** Both deviations were fixed in the `makeEmailLayers` factory function. Due to parallel execution with Plan 01, the fixes were incorporated into Plan 01's commits. This plan's commits focus exclusively on the test file changes.
